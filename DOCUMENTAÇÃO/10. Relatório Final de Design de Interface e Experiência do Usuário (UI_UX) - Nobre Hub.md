# Relatório Final de Design de Interface e Experiência do Usuário (UI/UX) - Nobre Hub

**Objetivo:** Fornecer um guia detalhado e completo para a implementação da interface do Nobre Hub, replicando a eficiência e a experiência de usuário de um CRM de alto nível como a Clint.

## 1. Fundamentos de Design e Identidade Visual

A interface deve ser **limpa**, **focada** e **intuitiva**, minimizando a carga cognitiva do usuário.

### 1.1. Paleta de Cores

A paleta deve ser funcional, utilizando cores para comunicar status e hierarquia.

| Uso | Cor (Exemplo Hex) | Função |
| :--- | :--- | :--- |
| **Primária (Ação)** | `#007bff` (Azul) | Botões de ação principal, links, indicadores de progresso. |
| **Secundária (Sucesso)** | `#28a745` (Verde) | Leads Fechados, Tarefas Concluídas, Alertas Positivos. |
| **Alerta (Urgência)** | `#dc3545` (Vermelho) | Leads Frios, Erros, Tarefas Atrasadas. |
| **Aviso (Atenção)** | `#ffc107` (Amarelo) | Leads sem Atividade, Status de Espera. |
| **Fundo (Base)** | `#f8f9fa` (Cinza Claro) | Fundo geral da aplicação, Sidebar. |
| **Texto (Principal)** | `#212529` (Preto Quase Total) | Leitura de conteúdo. |

### 1.2. Tipografia

Uma tipografia moderna e legível é crucial para a eficiência em um ambiente de trabalho.

*   **Família:** `Inter` ou `Roboto` (Sans-serif) - Priorizar legibilidade em telas pequenas e grandes.
*   **Hierarquia:**
    *   **Título Principal (H1):** 24px, Bold (Para títulos de módulo).
    *   **Título Secundário (H2):** 18px, Semi-Bold (Para títulos de seção dentro do modal).
    *   **Corpo do Texto:** 14px, Regular (Para mensagens e notas).
    *   **Texto de Status:** 12px, Medium (Para tags e informações de card).

## 2. Arquitetura de Informação e Fluxos de Usuário

A arquitetura deve refletir a filosofia "Cada um no seu quadrado" (SDR, Closer, Produção).

### 2.1. Mapeamento de Módulos (Sidebar)

| Módulo | Público-Alvo | Função Principal |
| :--- | :--- | :--- |
| **Dashboard** | Gerentes, Admin | Visão geral de KPIs (BI, Funil, Performance). |
| **Kanban** | SDRs, Closers | Gestão do Pipeline de Vendas (Core CRM). |
| **Inbox** | SDRs, Closers | Atendimento Omnichannel (WhatsApp). |
| **Produção** | Produção, Gerentes | Gestão do Pipeline de Entregas (Substitui Trello). |
| **Equipe** | Admin, Gerentes | Gestão de Usuários, Roles e Permissões. |
| **Automações** | Admin | Criação e Edição de Playbooks (Flow Builder). |

### 2.2. Fluxo de Usuário Crítico: Gestão de Lead

O fluxo deve ser otimizado para a menor quantidade de cliques possível.

1.  **Entrada:** Lead aparece no **Inbox** (Fila de Espera) ou no **Kanban** (Primeira Etapa).
2.  **Atribuição:** Agente clica em "Assumir Conversa" no Inbox.
3.  **Ação:** Agente abre o **Chat** (Visão 360º).
4.  **Qualificação:** Agente segue o **Playbook** (Barra Lateral do Chat).
5.  **Mudança de Etapa:** Agente clica no botão "Mover para Próxima Etapa" (Barra Lateral do Chat).
6.  **Fechamento:** Lead movido para "Fechado" no Kanban, ativando o **Playbook de Transição Vendas → Produção**.

## 3. Detalhamento de Componentes e Micro-Interações

### 3.1. Componente: Card do Lead (Kanban)

| Elemento | Detalhe de UI/UX | Micro-Interação |
| :--- | :--- | :--- |
| **Tempo na Etapa** | Ícone de relógio + texto (ex: "3 dias"). Cor muda para **Alerta** após 48h. | Tooltip ao passar o mouse com a data exata de entrada na etapa. |
| **Tags** | Tags coloridas (ex: "Alto Valor", "Instagram"). | Clique na tag para filtrar o Kanban. |
| **Drag-and-Drop** | Sombra (drop shadow) sutil no card ao ser arrastado. | Animação de "bounce" ao soltar o card na nova coluna. |
| **Ação Rápida** | Ícone de "Nova Tarefa" no canto do card. | Abre um mini-modal de criação de `Activity` sem abrir a Visão 360º. |

### 3.2. Componente: Chat (Inbox Unificada)

| Elemento | Detalhe de UI/UX | Micro-Interação |
| :--- | :--- | :--- |
| **Transcrição de Áudio** | Ícone de "Play" com texto "Áudio Transcrito por IA". | Ao clicar no ícone, o texto da transcrição aparece abaixo do balão de áudio. |
| **Barra Lateral CRM** | Fixo, com botão de "Expandir/Minimizar". | Ao mudar o `PipelineStage` na barra lateral, uma notificação de sucesso aparece no topo do chat. |
| **Caixa de Entrada** | Botão de "Templates" com atalho de teclado (ex: `/template`). | Abre um menu de busca rápida por templates de resposta. |

### 3.3. Componente: Flow Builder (Automações)

O `FlowBuilder.tsx` deve ser um ambiente de design visual.

*   **Nós (Nodes):** Representam Gatilhos, Condições e Ações. Devem ser coloridos por tipo (ex: Amarelo para Gatilho, Azul para Ação, Laranja para Condição).
*   **Conexões (Edges):** Linhas que conectam os nós. Devem ser curvas para melhor estética e clareza.
*   **Lógica Condicional:** O nó de Condição deve ter duas saídas claramente rotuladas: **"Sim" (Verde)** e **"Não" (Vermelho)**.

## 4. Recomendações de Acessibilidade e Performance

*   **Performance:** Utilizar a virtualização de listas (ex: `react-window`) no Kanban e no Inbox para lidar com milhares de leads e mensagens sem lentidão.
*   **Acessibilidade (WCAG):** Garantir contraste de cores adequado (especialmente para textos de status) e navegação completa por teclado (Tab Index) para todos os componentes interativos.
*   **Responsividade:** O design deve ser *mobile-first* para o Inbox (atendimento em campo) e *desktop-first* para o Kanban e Dashboards (trabalho de escritório).
