# Guia de Interface e Experiência do Usuário (UI/UX) - Nobre Hub

Este guia detalha a estrutura da interface da Clint CRM, servindo como blueprint para a IA de desenvolvimento do Nobre Hub. O objetivo é replicar a **eficiência operacional** e a **clareza visual** que tornam a Clint um CRM de alta performance.

## 1. Estrutura de Navegação e Hierarquia Visual

A Clint utiliza um layout padrão de aplicações SaaS (Software as a Service) focado em **acesso rápido** e **visão centralizada**.

| Componente | Localização | Função | Princípio de UX |
| :--- | :--- | :--- | :--- |
| **Sidebar Principal** | Lateral Esquerda (Fixo) | Navegação entre módulos principais (Kanban, Inbox, Dashboards, Configurações). | **Acesso Imediato:** O usuário deve alternar entre os módulos com um clique. |
| **Barra Superior (Header)** | Topo (Fixo) | Ações globais (Busca Rápida, Notificações, Perfil do Usuário). | **Ações Globais:** Ferramentas que o usuário precisa em qualquer tela. |
| **Área de Conteúdo Principal** | Centro | Onde o trabalho é realizado (Kanban, Chat, Gráficos). | **Foco no Trabalho:** Máximo espaço para o conteúdo principal. |
| **Modal/Drawer** | Sobreposição (Lateral Direita) | Exibição de detalhes do Lead, criação de tarefas, configurações rápidas. | **Contexto Preservado:** Permite ao usuário interagir com detalhes sem sair da tela principal. |

## 2. Detalhamento dos Componentes-Chave

### 2.1. O Kanban (Pipeline Visual)

O Kanban é o coração do CRM e deve ser otimizado para o fluxo de trabalho.

*   **Colunas (PipelineStage):** Devem ser claramente separadas, com cores que representam o tipo de pipeline (ex: Azul para High Ticket, Verde para Low Ticket).
*   **Cards (Leads):** Devem ser informativos, mas concisos.
    *   **Informações Mínimas:** Nome do Lead, Valor Estimado, Tempo na Etapa (para gerar senso de urgência), Foto do Agente Atribuído.
    *   **Interação:** O *drag-and-drop* deve ser suave e a mudança de etapa deve ser instantânea.
*   **Ações de Coluna:** Botão para adicionar novo Lead no topo de cada coluna.

### 2.2. O Chat (Inbox Unificada)

A UX do chat deve ser focada em **contexto de vendas** e **eficiência operacional**.

| Área do Chat | Conteúdo | Função de Vendas |
| :--- | :--- | :--- |
| **Lista de Conversas** | Lateral Esquerda. Filtros claros: "Minhas", "Fila", "Fechadas". | **Organização:** Permite ao agente priorizar e gerenciar sua carga de trabalho. |
| **Área de Mensagens** | Centro. Exibição de balões de conversa (WhatsApp). | **Comunicação:** Suporte a texto, áudio (com transcrição), imagens. |
| **Barra Lateral do CRM** | Lateral Direita (Fixo). | **Contexto:** Exibe o `Lead.name`, `Lead.estimatedValue`, `PipelineStage` atual e um botão de **Ação Rápida** (ex: Mover Etapa). |
| **Caixa de Entrada** | Fundo. Botões para: Enviar Template, Anexar Mídia, Enviar Resposta Rápida. | **Eficiência:** Reduz o tempo de resposta e padroniza a comunicação. |

### 2.3. O Modal de Detalhes do Lead (Visão 360º)

Este modal (ou Drawer) é a central de informações.

*   **Estrutura em Abas:**
    *   **Aba 1: Detalhes:** Dados de contato, UTMs (`LeadSourceData`), valor, notas.
    *   **Aba 2: Linha do Tempo:** Log cronológico de `Message`, `Activity` e `AuditLog`.
    *   **Aba 3: Playbook/Tarefas:** Lista de `Activity` pendentes e o "Script da Etapa" atual.
*   **Ações de Playbook:** O script de vendas deve ser exibido de forma proeminente, com a opção de marcar tarefas como concluídas.

## 3. Instruções de Design e UX para a IA

Para instruir a IA de desenvolvimento (ou o time de frontend), os seguintes princípios devem ser seguidos:

1.  **Prioridade à Tipagem (TypeScript):** Todos os componentes de UI devem ser construídos com base nos tipos definidos no Schema Prisma Otimizado (v3.0) para garantir a consistência dos dados.
2.  **Princípio da Ação Imediata:** Sempre que um dado for exibido (ex: `Lead.phone`), ele deve ser clicável para iniciar uma ação (ex: Abrir WhatsApp Web).
3.  **Consistência Visual (TailwindCSS):** Utilizar o TailwindCSS para garantir um design limpo, moderno e responsivo, com ênfase em cores de status (ex: Vermelho para leads frios, Verde para leads quentes).
4.  **Componentização do Flow Builder:** O `FlowBuilder.tsx` deve ser um componente visual que traduz o JSON do `AutomationFlow.flowDefinition` em nós e arestas interativos (usando React Flow).
5.  **Micro-Interações:** Animações sutis (ex: ao arrastar um card no Kanban) para dar feedback ao usuário e tornar a experiência mais agradável.
